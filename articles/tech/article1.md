# 关于WSL2的内核定制与摄像头的使用

## 0基础配置：修改WSL2内核并使用内部摄像头（完整教程）

本文专为0基础新手编写，全程无专业术语堆砌，每一步均附具体命令、操作截图提示（文字描述替代）和注意事项，无需提前了解WSL2、内核、驱动等知识，跟着操作即可完成配置，实现WSL2（Ubuntu）调用电脑内部摄像头。

核心逻辑：WSL2默认内核不支持摄像头驱动，我们需要先安装WSL2、配置基础环境，再下载内核源码、修改驱动配置、重新编译内核，最后通过工具将电脑内部摄像头转发到WSL2，完成权限配置和测试，全程一步步推进，无跳步。

## 前置说明（0基础必看）

- 适用场景：Windows 10/11 系统，想在WSL2（推荐Ubuntu发行版）中使用电脑内置摄像头（如笔记本自带摄像头），用于开发、测试、视频采集等。

- 准备工具：电脑需联网，无需提前安装任何软件，全程跟着命令操作即可。

- 操作节奏：每一步执行完成后，确认无报错再进行下一步，避免跳过步骤导致失败。

- 关键提示：所有命令均需严格复制粘贴（避免手动输入出错），区分「Windows PowerShell」和「WSL2终端」，不要混淆操作窗口。

## 一、0基础安装WSL2（Ubuntu发行版）

如果你的电脑还没有安装WSL2，先执行这一步；如果已经安装WSL2（能打开Ubuntu终端），直接跳过这一步，进入第二步。

### 1.1 开启Windows相关功能（管理员权限操作）

1.  按下键盘「Win + X」，选择「Windows终端（管理员）」（Win10可能显示「PowerShell（管理员）」），打开后会弹出黑色窗口（这就是PowerShell）。

2.  复制以下命令，粘贴到PowerShell窗口中，按下回车（执行过程中不要关闭窗口，会自动开启相关功能）：

```powershell

wsl --install

```

3.  命令执行完成后，会提示「请重启电脑以完成安装」，关闭所有窗口，重启电脑（必须重启，否则WSL2无法生效）。

### 1.2 初始化WSL2（Ubuntu）

1.  电脑重启后，会自动弹出「Ubuntu」窗口，等待片刻（首次启动会自动解压安装，约5-10分钟）。

2.  安装完成后，会提示设置「用户名」和「密码」（用户名建议用英文小写，密码输入时不显示，输入完成按回车即可，无需记住，后续可重置）。

3.  用户名和密码设置完成后，Ubuntu终端就打开了（黑色窗口，前缀是「用户名@电脑名:~$」），这就是我们后续操作的主要窗口。

### 1.3 验证WSL2安装成功

在Ubuntu终端中，复制粘贴以下命令，按下回车：

```bash

uname -r

```

如果输出类似「5.15.167.4-microsoft-standard-WSL2」的内容，说明WSL2安装成功，进入下一步；若报错，重新执行第一步的「wsl --install」命令，重启电脑后再试。

## 二、配置基础环境（必做，为内核修改做准备）

这一步主要安装「内核编译依赖工具」和「USB设备转发工具」，是后续修改内核、调用摄像头的基础，全程复制命令执行即可。

### 2.1 更新WSL2（Ubuntu）系统

在Ubuntu终端中，复制粘贴以下命令（分两行执行，先执行第一行，无报错再执行第二行）：

```bash

sudo apt update

sudo apt upgrade -y

```

说明：「sudo」表示管理员权限，执行时会提示输入你第一步设置的密码（输入时不显示，输完按回车）；「-y」表示自动确认安装，无需手动输入「y」。

### 2.2 安装内核编译依赖工具

WSL2默认没有安装「编译内核」所需的工具，复制以下完整命令，粘贴到Ubuntu终端，按下回车，等待安装完成（约5-10分钟，取决于网络速度）：

```bash

sudo apt install -y build-essential flex bison dwarves libssl-dev libelf-dev libncurses-dev pkg-config git

```

无需理解这些工具的作用，安装完成即可，后续编译内核会自动调用。

### 2.3 安装USB设备转发工具（usbipd-win）

内部摄像头是电脑的USB设备，WSL2无法直接识别，需要通过「usbipd-win」工具转发，操作如下：

1.  关闭Ubuntu终端，重新打开「Windows终端（管理员）」（Win + X 选择）。

2.  复制以下命令，粘贴到PowerShell窗口，按下回车，安装工具：

```powershell

winget install usbipd

```

3.  安装完成后，关闭PowerShell，重新打开Ubuntu终端，继续下一步。

## 三、下载WSL2内核源码（核心步骤1）

我们需要下载WSL2的内核源码，后续修改「摄像头驱动配置」，再重新编译内核，步骤如下（全程在Ubuntu终端操作）。

### 3.1 查看当前WSL2内核版本

复制以下命令，粘贴到Ubuntu终端，按下回车，记录输出的内核版本（如「5.15.167.4」，无需记完整，记前面的数字即可）：

```bash

uname -r

```

示例输出：5.15.167.4-microsoft-standard-WSL2 → 记住「5.15.167.4」这个版本号。

### 3.2 创建内核源码目录并下载源码

复制以下命令（分4行，依次粘贴执行，每执行一行按回车，无报错再执行下一行）：

```bash

# 1. 创建目录（用于存放内核源码，避免混乱）

mkdir -p ~/wsl-kernel && cd ~/wsl-kernel

# 2. 替换版本号（将后面的数字换成你刚才记录的版本号，如5.15.167.4）

TAGVERNUM=5.15.167.4

TAGVER=linux-msft-wsl-${TAGVERNUM}

# 3. 下载内核源码（无需修改，直接执行）

sudo git clone --depth 1 -b ${TAGVER} https://github.com/microsoft/WSL2-Linux-Kernel.git ${TAGVERNUM}-microsoft-standard

# 4. 进入源码目录（后续操作都在这里进行）

cd ${TAGVERNUM}-microsoft-standard

```

注意：如果下载速度很慢（git克隆卡住），可以这样操作：

1.  打开浏览器，访问链接：https://github.com/microsoft/WSL2-Linux-Kernel/releases

2.  在页面中找到与你内核版本匹配的压缩包（如「linux-msft-wsl-5.15.167.4.tar.gz」），点击下载。

3.  下载完成后，将压缩包拖到「Ubuntu」终端中（会自动复制到当前目录），然后执行解压命令（替换压缩包名称为你下载的名称）：

```bash

tar vxf ./linux-msft-wsl-5.15.167.4.tar.gz

cd ./linux-msft-wsl-5.15.167.4

```

解压完成后，继续下一步。

## 四、修改内核配置（启用摄像头驱动，核心步骤2）

这一步是最关键的一步：WSL2默认内核没有启用摄像头相关驱动，我们需要通过「菜单式配置」，手动启用驱动，步骤如下（全程在Ubuntu终端操作，不要关闭终端）。

### 4.1 打开内核配置界面

复制以下命令，粘贴到Ubuntu终端，按下回车，会弹出一个蓝色的菜单界面（内核配置界面）：

```bash

sudo make menuconfig KCONFIG_CONFIG=Microsoft/config-wsl

```

操作提示：菜单界面用「方向键」上下导航，「回车键」进入子菜单，「空格键」切换配置状态（按一次空格是「M」，按两次是「*」，按三次是空白，我们需要设为「*」，表示启用驱动并内置到内核）。

### 4.2 逐一启用摄像头相关驱动（0基础无脑操作）

按照以下路径，依次导航并设置，每一步都要确认设置为「*」，不要遗漏：

1.  用方向键选中「Device Drivers」（设备驱动），按回车键进入子菜单；

2.  选中「Multimedia support」（多媒体支持），按空格键设为「*」，再按回车键进入子菜单；

3.  选中「Filter media drivers」，按空格键设为「*」；

4.  选中「Autoselect ancillary drivers (tuners, sensors, i2c, spi, frontends)」，按空格键设为「*」；

5.  按「ESC键」退回上一级菜单（Device Drivers），选中「Media device types」（媒体设备类型），按回车键进入；

6.  选中「Cameras and video grabbers」（摄像头和视频采集器），按空格键设为「*」；

7.  按「ESC键」退回上一级菜单，选中「Video4Linux options」（V4L2选项），按回车键进入；

8.  选中「V4L2 sub-device userspace API」，按空格键设为「*」；

9.  按「ESC键」退回上一级菜单，选中「Media drivers」（媒体驱动），按回车键进入；

10. 选中「Media USB Adapters」（USB媒体适配器），按空格键设为「*」，按回车键进入；

11. 选中「USB Video Class (UVC)」，按空格键设为「*」；

12. 选中「GSPCA based webcams」，按空格键设为「*」（兼容所有品牌的内部摄像头）；

13. 按「ESC键」退回上一级菜单（Device Drivers），选中「USB support」（USB支持），按回车键进入；

14. 选中「USB/IP support」（USB/IP支持），按空格键设为「*」，进入子菜单后，将所有选项都设为「*」；

15. 所有配置完成后，按「ESC键」退回最外层菜单，用方向键选中「Save」（保存），按回车键，再按回车键确认保存（默认保存路径，无需修改）；

16. 选中「Exit」（退出），按回车键，逐层退出配置界面，回到Ubuntu终端。

注意：如果不小心选错，可重新执行「sudo make menuconfig KCONFIG_CONFIG=Microsoft/config-wsl」，重新配置即可。

## 五、编译并替换WSL2内核（核心步骤3）

配置完成后，我们需要「编译内核」（将修改后的驱动配置生效），再替换WSL2默认的内核，步骤如下（全程在Ubuntu终端操作，耗时较长，耐心等待）。

### 5.1 编译内核

复制以下命令，粘贴到Ubuntu终端，按下回车，开始编译内核：

```bash

sudo make -j$(nproc) KCONFIG_CONFIG=Microsoft/config-wsl

```

说明：

- 编译速度取决于电脑CPU核心数，一般需要10-30分钟，期间不要关闭终端，不要操作电脑（避免卡顿）。

- 编译成功的标志：终端最后会输出「Kernel: arch/x86/boot/bzImage is ready」，如果没有报错，说明编译成功；若报错，重新检查第四步的驱动配置，确保所有选项都设为「*」，再重新编译。

### 5.2 安装编译好的内核模块

编译完成后，复制以下命令，粘贴到终端，按下回车，安装内核模块（驱动生效的关键）：

```bash

sudo make modules_install headers_install

```

安装完成后，无报错即可进入下一步。

### 5.3 替换WSL2默认内核

我们需要将编译好的内核，复制到Windows系统中，再配置WSL2使用新内核，步骤如下：

1.  复制以下命令，粘贴到Ubuntu终端，将内核镜像复制到Windows的C盘（无需手动打开C盘，命令会自动挂载）：

```bash

# 创建C盘挂载目录（若已挂载可跳过）

sudo mkdir -p /mnt/c/vmlinux

# 复制编译好的内核镜像到C盘

sudo cp vmlinux /mnt/c/vmlinux/

```

2.  关闭Ubuntu终端，打开「文件资源管理器」，进入「C:\Users\你的用户名」目录（如「C:\Users\Admin」，「你的用户名」是Windows的用户名）。

3.  在该目录下，右键「新建」→「文本文档」，命名为「.wslconfig」（注意前面有个点，后缀不要留.txt，若看不到后缀，可在文件资源管理器中开启「显示文件扩展名」）。

4.  打开「.wslconfig」文件，粘贴以下内容，保存并关闭：

```yaml

[wsl2]

kernel=C:\\vmlinux\\vmlinux

```

5.  打开「Windows终端（管理员）」，复制以下命令，粘贴执行，重启WSL2，使新内核生效：

```powershell

wsl --shutdown

```

6.  重新打开Ubuntu终端，执行以下命令，验证内核是否替换成功：

```bash

uname -r

```

若输出的内核版本与你编译的版本一致（如5.15.167.4），说明内核替换成功；若不一致，检查「.wslconfig」文件的路径是否正确，重新执行步骤5.3。

## 六、转发内部摄像头到WSL2（核心步骤4）

内核修改完成后，需要将电脑的内部摄像头，通过之前安装的「usbipd-win」工具，转发到WSL2，让WSL2识别到摄像头，步骤如下（Windows和Ubuntu配合操作）。

### 6.1 查看摄像头的BUSID（Windows操作）

1.  打开「Windows终端（管理员）」，复制以下命令，粘贴执行，查看电脑上所有USB设备：

```powershell

usbipd list

```

2.  在输出结果中，找到「Integrated Camera」（内部摄像头），记录它的「BUSID」（格式为「x-x」，如「1-7」，前面的数字是总线号，后面是设备号）。

示例：USB Camera (VID:04f2 PID:b604) → BUSID: 1-7 → 记住「1-7」。

### 6.2 绑定并转发摄像头（Windows操作）

在PowerShell窗口中，复制以下命令（替换「1-7」为你记录的BUSID），粘贴执行，绑定摄像头：

```powershell

usbipd bind --busid 1-7

```

绑定成功后，复制以下命令，粘贴执行，将摄像头转发到WSL2：

```powershell

usbipd attach --wsl --busid 1-7

```

提示：转发成功后，Windows的摄像头会暂时被WSL2占用，关闭WSL2后会自动释放。

### 6.3 配置WSL2摄像头权限（Ubuntu操作）

转发完成后，WSL2已识别到摄像头，但默认没有访问权限，复制以下命令（分3行执行），配置权限：

```bash

# 将当前用户添加到摄像头用户组

sudo usermod -a -g video $USER

# 赋予摄像头读写权限

sudo chgrp video /dev/video0 /dev/video1

sudo chmod 660 /dev/video0 /dev/video1

```

配置完成后，退出Ubuntu终端，重新打开（权限生效）。

## 七、测试摄像头是否正常工作

完成以上所有步骤后，我们用最简单的方法，测试摄像头是否能正常使用，全程在Ubuntu终端操作。

### 7.1 确认摄像头已被识别

复制以下命令，粘贴到Ubuntu终端，按下回车：

```bash

ls /dev/video*

```

若输出「/dev/video0 /dev/video1」，说明摄像头已被WSL2识别；若没有输出，重新执行第六步的转发和权限配置。

### 7.2 安装测试工具，查看实时画面

复制以下命令，安装简单的摄像头测试工具（guvcview）：

```bash

sudo apt install -y guvcview

```

安装完成后，复制以下命令，启动测试工具：

```bash

guvcview

```

启动后，会弹出一个窗口，若能看到摄像头采集的实时画面（自己的脸），说明配置成功，WSL2可以正常使用内部摄像头了！

关闭测试工具：点击窗口右上角的「X」，或在终端中按「Ctrl + C」。

## 八、0基础常见问题排查（必看）

新手操作中可能遇到以下问题，直接对应解决即可，无需复杂排查：

### 问题1：编译内核报错，提示「no rule to make target」

原因：第四步的驱动配置遗漏，或第二步的依赖工具未安装完整。

解决：重新执行第二步的「安装内核编译依赖工具」命令，再重新执行第四步的驱动配置，确保所有选项都设为「*」，重新编译。

### 问题2：ls /dev/video* 无输出，摄像头未被识别

原因：摄像头未转发成功，或BUSID选错。

解决：重新执行第六步，确认BUSID与摄像头对应，重新绑定并转发，再配置权限。

### 问题3：guvcview启动后黑屏，无画面

原因：Windows的摄像头被其他软件占用（如微信、相机），或权限未生效。

解决：关闭Windows中所有使用摄像头的软件，重新执行转发命令；退出Ubuntu终端，重新打开，再启动guvcview。

### 问题4：重启电脑后，摄像头无法使用

原因：重启后，摄像头转发失效，需要重新转发。

解决：重新执行第六步的「绑定并转发摄像头」步骤，无需重新编译内核。

## 九、总结（0基础总结）

全程按照「安装WSL2 → 配置基础环境 → 下载内核源码 → 修改驱动配置 → 编译替换内核 → 转发摄像头 → 测试使用」的步骤操作，0基础也能成功配置。

核心重点：第四步的驱动配置（所有选项设为「*」）、第五步的内核编译（耐心等待）、第六步的摄像头转发和权限配置，这三步不出错，就能成功调用内部摄像头。

后续使用：每次重启电脑后，只需重新执行「第六步：转发内部摄像头到WSL2」，即可正常使用摄像头；若需要卸载WSL2，执行「wsl --unregister Ubuntu」命令即可。
